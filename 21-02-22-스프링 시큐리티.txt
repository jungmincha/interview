sql injection

#{id}를 쓸 때 입력값이 3이라면 오라클에 바인딩 될 때 ? 로 넘어가고 출력될 때 3으로 표시된다.

 preparedStatement 방식이며 재사용 가능하다.

 중간에 ? 로 바인딩되기 때문에 보안이 강하며 sql injection 공격에 강하다.


용도 : 자주 바뀌는 값을 전달할 때, 많은 값을 전달할 때


${id}를 쓸 때 입력값이 3이라면 오라클에 바인딩 될 때 3으로 넘어가고 출력될 때 3으로 표시된다.

statement 방식이며 재사용 불가능하다. 새로운 쿼리로 인식하기 때문에 속도가 좀 느리다. 

입력값이 그대로? 넘어가기 때문에 보안에 취약하다.


용도 : 자주 바뀌지 않는 값을 전달할때(테이블, 컬럼명)

sql injection : 클라이언트의 입력값을 조작하여 서버의 데이터베이스를 공격할 수 있는 공격방식
